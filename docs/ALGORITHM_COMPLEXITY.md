# Вычислительная сложность алгоритмов покрытия

Обозначения:

- `N` — число свободных и занятых клеток дискретной карты.
- `F` — число свободных клеток.
- `R` — число роботов.
- `W` — сторона локального окна ML-модели.
- `C` — число входных каналов модели.
- `K` — число кандидатов цели, рассматриваемых для каждого робота.
- `H` — горизонт короткого offline rollout при генерации обучающих меток.

## BaselineVoronoi

На каждом шаге алгоритм просматривает карту и распределяет непосещённые клетки по ближайшему роботу. Для каждой свободной непосещённой клетки вычисляется расстояние до каждого робота.

Оценка одного шага:

```text
O(F * R + R * F_r)
```

где `F_r` — число клеток в регионе конкретного робота. В худшем случае это можно считать как `O(F * R)`.

Плюсы: простая логика, хорошая устойчивость, естественное разделение области.  
Минусы: каждый шаг требует повторного просмотра карты, нет обучаемой адаптации к динамической сцене.

## BaselineFrontier

Алгоритм формирует список непосещённых клеток и выбирает ближайшую frontier/непосещённую цель.

Оценка одного шага:

```text
O(F + R * F)
```

В текущей реализации для каждого робота может просматриваться список кандидатов, поэтому практическая сложность близка к `O(R * F)`.

## STC и DARP

STC строит структурированный маршрут покрытия на клеточной карте. Для одной карты построение маршрута обычно линейно или почти линейно относительно числа клеток:

```text
O(F)
```

DARP сначала решает задачу распределения территории между роботами, затем строит маршруты в подзадачах. Практическая сложность зависит от числа итераций балансировки:

```text
O(I * F * R + R * F_r)
```

где `I` — число итераций перераспределения. В литературе mCPP в общем виде рассматривается как сложная комбинаторная задача; DARP снижает практическую сложность за счёт декомпозиции области.

## Step-level ML-guided

Старая ML-постановка выбирала один из `N/S/E/W/stop` по локальному окну.

Оценка инференса для одного робота:

```text
O(W^2 * C * P)
```

где `P` — условный коэффициент числа свёрточных фильтров. Для `R` роботов:

```text
O(R * W^2 * C * P)
```

Проблема этой постановки в том, что микрошаг плохо выражает цель покрытия. Модель может научиться локальным действиям, но не решает, какую область выгоднее покрывать следующей.

## ML-goal policy

Новая постановка выбирает локальную цель/frontier cell по многоканальному окну. Один forward pass возвращает heatmap целей.

Оценка инференса:

```text
O(R * W^2 * C * P)
```

После получения heatmap выбираются top-k кандидатов:

```text
O(R * W^2 log K)
```

или `O(R * W^2)` при линейном top-k.

## Ranked ML-goal + Allocation

Расширенная версия добавляет оценку top-k целей и распределение целей между роботами.

Скоринг кандидатов:

```text
O(R * K)
```

Greedy allocation:

```text
O(R * K log(R * K))
```

Hungarian assignment для матрицы `R x K`:

```text
O(max(R, K)^3)
```

Для дипломной сцены с 8 роботами и небольшим `K` эта стоимость приемлема, а выигрыш заключается в снижении overlap: несколько роботов реже выбирают одну и ту же область.

## Guardrail

Guardrail проверяет безопасность и прогресс выбранных целей:

```text
O(R * K)
```

Если цель небезопасна, находится вне карты, попадает в препятствие или приводит к стагнации, используется fallback (`BaselineVoronoi`):

```text
O(F * R)
```

## Offline best-of-k label generation

Для более сильного обучения можно строить метки не из одного действия учителя, а через offline выбор лучшей цели из `K` кандидатов:

```text
O(M * R * K * H)
```

где `M` — число обучающих состояний, `H` — короткий горизонт оценки. Это дороже, но выполняется offline и позволяет обучать модель на целевой функции покрытия:

```text
score = new_coverage - alpha * distance - beta * risk - gamma * overlap
```

## Итоговая сложность одного шага идеальной системы

Для ranked ML-goal с greedy allocation:

```text
O(N + R * W^2 * C * P + R * W^2 + R * K log(R * K) + R * K)
```

При фиксированных `W`, `K`, `C` и небольшом числе роботов основная часть остаётся линейной по размеру карты:

```text
O(N)
```

Это важно для дипломного вывода: система масштабируется практически линейно по сетке карты, а ML-инференс и allocation дают контролируемую добавку.
